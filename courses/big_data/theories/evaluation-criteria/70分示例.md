### 1.背景分析

传统的消费者价格指数(CPI)依赖人工调研和月度更新，存在更新慢、数据粗、覆盖不足等问题。随着电商平台大量数据的涌现，利用其高频率、广覆盖和高精度的优势，可以构建更及时、精细的价格指数，以更敏锐地反映市场变动。本项目正是基于这一背景，借助云服务和大数据技术，构建一套高效、自动化的电商价格指数计算体系。

### 2.概念

#### 2.1重要概念

如何从日常收集的在线价格数据构建价格指数

重点解释:3.1节(在线价格指数的计算方法)

3.1节详细描述了如何从日常收集的在线价格数据构建价格指数，具体步骤如下:

##### 1.分类内价格变化的几何平均

价格变化首先在产品类别内计算几何平均。公式为:

$$R_{t, t-1}^j=\prod_i\left(\frac{p_t^i}{p_{t-1}^i}\right)^{1/ n_{j, t}}$$

*   $p_{t}^{i}$ :第t天商品i的价格;
*   $n_{j, t}$ :第t天类别j中的商品数量。
*   几何平均的选用是为了减少极端价格波动的影响,更贴近消费者实际购买体验。

##### 2.分类指数的链式累积

每个类别的价格指数通过逐日链式累积计算:

$$I_t^j= R_{1,0}^j\cdot R_{2,1}^j\cdots R_{t,t-1}^j$$

*   这一过程将每日的相对价格变化转化为长期指数,反映类别内价格的整体趋势。

##### 3.跨类别的加权算术平均

最终超市价格指数是各分类指数的加权算术平均:

$$ S_t=\sum_j\frac{w^j}{W} I_t^j$$

*   $w^j$ :官方CPI中类别 j的权重;
*   W:样本中包含的所有类别权重之和。
*   算术平均的选用是为了与官方CPI的聚合方法一致,确保可比性。

##### 4.高频数据的处理与稳健性

*   动态篮子:允许商品随时间进入或退出市场,仅在有价格变化时纳入计算(新商品次日生效,退出商品立即剔除)。
*   数据填补:短期价格空缺(如缺货或爬虫失败)采用“最后可用价格”填补,与BLS的“单元格相对调整法”结果一致。
*   分类与权重:通过产品URL映射到标准化官方分类,确保权重分配基于实际消费结构。

##### 5.月度与年度通胀率的计算

*   月度通胀率:过去30天指数均值与前30天均值的百分比变化;
*   年度通胀率:过去30天指数均值与去年同期均值的百分比变化。
*   该方法与阿根廷统计局的CPI计算方式一致,避免了对数差分可能引入的偏差。

*   作用:
    *   动态追踪价格趋势:每日更新,避免依赖固定基期(如传统CPI以某年为基准)。
    *   消除基准偏差:例如,若基期价格异常低,链式方法可减少其对后续指数的影响。

*   示例:
    若某类别第1天指数为100,第2天上涨5%(指数=105),第3天上涨3%(指数=1051.03=108.15)，依此类推。

*   作用:
    *   反映实际消费结构:高权重类别(如食品)对总指数影响更大。
    *   与官方方法一致:确保在线指数与CPI可比。

*   示例:
    若食品类权重30%，电子产品权重10%，则食品价格上涨10%对总指数的贡献为3%(=30%x10%)，电子产品上涨20%贡献2%(=10%x20%)。

### 3.系统架构

#### 3.1数据采集(提供)

*   来源:主流电商平台(如淘宝、京东、拼多多等)商品价格信息
*   方法:API接口或网页爬虫
*   内容:商品名、价格、品类、品牌、时间戳等元数据

#### 3.2数据存储

*   对象存储(如阿里OSS、腾讯COS)
    *   用于存放原始数据(CSV/JSON等)和处理中间结果
*   ClickHouse用于实时数据分析的开源列式数据库
    *   高性能存储处理后的结构化数据(按日价格、分类、品牌等)

Clickhouse创建零时表

```sql
CREATE TABLE IF NOT EXISTS raw_product_price_temp(
    platform String,
    category_raw String,
    product_name String,
    brand_raw String,
    price Float64,
    timestamp DateTime,
    url String,
    status String
) ENGINE= MergeTree()
ORDER BY(timestamp, platform);
```

### 数据清洗
*   标准化分类名
*   提取品牌名(brand)
*   异常价格过滤(例如价格为0、过高)

#### 写入表

```sql
CREATE TABLE IF NOT EXISTS clean_product_price(
    platform String,
    category String,
    product_name String,
    brand String,
    price Float64,
    capture_date Date,
    url String
)ENGINE=MergeTree()
```

### 4.数据设置

#### 原始数据采集
| 字段名        | 类型     | 描述                          |
|---------------|----------|-------------------------------|
| id            | String   | 唯一标识(UUID)                |
| platform      | String   | 来源平台(如淘宝、京东)        |
| category_raw  | String   | 原始类目名称                  |
| product_name  | String   | 商品标题                      |
| brand_raw     | String   | 原始品牌名(可能不规范)        |
| price         | Float    | 当前价格(单位元)              |
| timestamp     | DateTime | 抓取时间                      |
| url           | String   | 商品链接(可选)                |
| status        | String   | 抓取状态(成功、失败等)        |

#### 标准商品价格
| 字段名       | 类型     | 描述                                |
|--------------|----------|-------------------------------------|
| id           | String   | 唯一标识                            |
| platform     | String   | 平台                                |
| category     | String   | 标准类目(如手机、家电、服装)        |
| product_name | String   | 商品名(清洗后)                      |
| brand        | String   | 品牌名(标准化)                      |
| price        | Float    | 商品价格                            |
| price_unit   | String   | 单位(元/件、元/斤等)                |
| capture_date | Date     | 抓取日期(按日归档)                  |
| weight       | Float    | 商品权重(用于加权平均,可选)         |

#### 价格指数结果表
| 字段名        | 类型     | 描述                                |
|---------------|----------|-------------------------------------|
| index_date    | Date     | 指数日期(每日)                      |
| category      | String   | 类别(如手机、家电)                  |
| index_value   | Float    | 当日价格指数值                      |
| index_type    | String   | 指数类型(如拉氏指数、帕氏指数)      |
| base_date     | Date     | 基期日期(通常为项目起始日)          |
| yoy_change    | Float    | 同比变化率(%)                       |
| mom_change    | Float    | 环比变化率(%)                       |

### 5.数据处理
*   主要包含清洗、去重、标准化分类等预处理操作
*   价格指数构建方法:
    *   简单平均法、加权平均法
    *   可选基期(如拉氏指数、帕氏指数)
*   实现方式:
    *   Python脚本定时运行,结合Pandas+ SQL查询
    *   并发调度可用 Celery+Redis或云函数(如阿里云函数计算)
